Loops

Loops in Java are used to execute a block of code repeatedly until a specified condition is met. They are essential for repetitive tasks and allow for efficient coding.

There are three main types of loops in Java:

  1. For Loop: Executes a block of code a specified number of times.
  2. While Loop: Executes a block of code as long as a specified condition is true.
  3. Do-While Loop: Similar to the while loop, but it executes the block of code at least once, then repeats the loop as long as the specified condition is true.

1. For Loop

The for loop is commonly used when the number of iterations is known in advance.

Example:

for (int i = 0; i < 5; i++) {
                System.out.println("Iteration: " + i);
            }

This loop will print "Iteration: 0" to "Iteration: 4".

2. While Loop

The while loop repeats a block of code as long as a specified condition is true.

Example:

int i = 0;
            while (i < 5) {
                System.out.println("Iteration: " + i);
                i++;
            }

This loop will also print "Iteration: 0" to "Iteration: 4".

3. Do-While Loop

The do-while loop is similar to the while loop, but it always executes the block of code at least once, even if the condition is false.

Example:

int i = 0;
            do {
                System.out.println("Iteration: " + i);
                i++;
            } while (i < 5);

This loop will also print "Iteration: 0" to "Iteration: 4".

If-Else Statements

In Java, the if statement is used to execute a block of code if a specified condition is true. Optionally, the else statement can be used to execute a block of code if the condition is false.

Example:

int num = 10;
            if (num % 2 == 0) {
                System.out.println("Even number");
            } else {
                System.out.println("Odd number");
            }

This code checks if the variable num is even or odd and prints the result accordingly.

Nested If-Else Statements

You can also use nested if-else statements to check multiple conditions.

Example:

int age = 25;
            if (age < 18) {
                System.out.println("You are a minor");
            } else if (age < 65) {
                System.out.println("You are an adult");
            } else {
                System.out.println("You are a senior citizen");
            }

This code checks the age of a person and prints the appropriate message based on their age.

Switch Statement

The switch statement is used to select one of many code blocks to be executed.

Example:

int day = 3;
            switch (day) {
                case 1:
                    System.out.println("Monday");
                    break;
                case 2:
                    System.out.println("Tuesday");
                    break;
                // Add cases for other days
                default:
                    System.out.println("Invalid day");
            }

This code prints the name of the day based on the value of the day variable.

Functions (Methods)

In Java, functions are called methods. They are blocks of code that perform a specific task and are defined within a class. Methods provide modularity and reusability in your code.

Example:

public class Example {
                // Method definition
                public void greet() {
                    System.out.println("Hello!");
                }
            
                // Main method
                public static void main(String[] args) {
                    Example obj = new Example();
                    obj.greet(); // Method invocation
                }
            }

This code defines a method named greet inside a class named Example. The greet method prints "Hello!". The method is then invoked in the main method.

Method Parameters

Methods can take parameters, which are values passed to the method when it is called. Parameters are specified within the parentheses after the method name.

Example:

public class Example {
                // Method definition with parameters
                public void add(int a, int b) {
                    int sum = a + b;
                    System.out.println("Sum: " + sum);
                }
            
                // Main method
                public static void main(String[] args) {
                    Example obj = new Example();
                    obj.add(5, 3); // Method invocation with arguments
                }
            }

This code defines a method named add that takes two parameters a and b. The method calculates the sum of the parameters and prints it. The method is then invoked with arguments 5 and 3.

Return Values

Methods can also return a value after execution. The return type of the method specifies the type of value returned by the method.

Example:

public class Example {
                // Method definition with return value
                public int square(int num) {
                    return num * num;
                }
            
                // Main method
                public static void main(String[] args) {
                    Example obj = new Example();
                    int result = obj.square(4); // Method invocation with return value
                    System.out.println("Square: " + result);
                }
            }

This code defines a method named square that takes an integer parameter num and returns the square of that number. The method is then invoked, and the returned value is printed.

Data Types

In Java, data types specify the type of data that a variable can hold. Java has two main categories of data types: primitive data types and reference data types.

Primitive Data Types

Primitive data types represent single values and are predefined by the language. They are used to store simple values such as numbers and characters.

  • int: Used to store integer numbers (whole numbers).
  • double: Used to store floating-point numbers (numbers with decimal points).
  • boolean: Used to store true or false values.
  • char: Used to store single characters.

Example:

int age = 25;
            double price = 10.99;
            boolean isJavaFun = true;
            char grade = 'A';

Reference Data Types

Reference data types are used to store references (memory addresses) to objects. They do not store the actual data but rather a reference to where the data is stored in memory.

Example:

String name = "Java";
            Integer number = new Integer(5);

Here, name is a reference to a String object, and number is a reference to an Integer object.

Variables

In Java, variables are used to store data values. They provide a way to label and reference memory locations in a program.

Variable Declaration

Before using a variable, it must be declared with a specified data type. Variable declaration allocates memory for the variable and associates a name with it.

Example:

int age; // Declaration of an integer variable named 'age'
            double price; // Declaration of a double variable named 'price'
            String name; // Declaration of a String variable named 'name'

Variable Initialization

Initialization is the process of assigning a value to a variable. A variable can be initialized at the time of declaration or later in the program.

Example:

int age = 25; // Declaration and initialization of an integer variable 'age'
            double price = 10.99; // Declaration and initialization of a double variable 'price'
            String name = "Java"; // Declaration and initialization of a String variable 'name'

Variable Scope

The scope of a variable defines where the variable can be accessed in the program. Variables can have different scopes depending on where they are declared.

Example:

public class Example {
                // Global variable accessible throughout the class
                int globalVar = 10;
            
                public void method() {
                    // Local variable accessible only within this method
                    int localVar = 20;
                    System.out.println(globalVar); // Accessing globalVar
                    System.out.println(localVar); // Accessing localVar
                }
            
                public static void main(String[] args) {
                    Example obj = new Example();
                    System.out.println(obj.globalVar); // Accessing globalVar outside the class
                    // System.out.println(obj.localVar); // Error: localVar is not accessible here
                }
            }

In this example, globalVar is accessible throughout the class, while localVar is only accessible within the method method.

Arrays

In Java, an array is a data structure that stores a fixed-size sequential collection of elements of the same type. Arrays provide a way to store multiple values under a single variable name.

Array Declaration

To declare an array in Java, you specify the type of elements the array will hold and the array's name, followed by square brackets ([]).

Example:

// Declaration of an integer array named 'numbers'
            int[] numbers;

Array Initialization

Arrays can be initialized with values at the time of declaration or later in the program.

Example:

// Declaration and initialization of an integer array with 3 elements
            int[] numbers = {1, 2, 3};

Accessing Array Elements

Array elements are accessed using an index, which starts from 0 for the first element and increments by 1 for each subsequent element.

Example:

int[] numbers = {1, 2, 3};
            System.out.println(numbers[0]); // Accessing the first element (prints 1)
            System.out.println(numbers[1]); // Accessing the second element (prints 2)
            System.out.println(numbers[2]); // Accessing the third element (prints 3)

Array Length

You can find the length of an array using the length property.

Example:

int[] numbers = {1, 2, 3};
            System.out.println("Length of the array: " + numbers.length); // Prints the length of the array (3)

Iterating Through an Array

You can use loops to iterate through the elements of an array.

Example:

int[] numbers = {1, 2, 3};
            for (int i = 0; i < numbers.length; i++) {
                System.out.println("Element at index " + i + ": " + numbers[i]);
            }

This code will print each element of the array along with its index.

Classes and Objects

In Java, a class is a blueprint for creating objects. An object is an instance of a class. Classes define the properties and behaviors (methods) of objects.

Class Declaration

To declare a class in Java, you use the class keyword followed by the class name.

Example:

public class Car {
                // Class definition
            }

Object Creation

To create an object of a class, you use the new keyword followed by the class name and parentheses (if there are no constructor arguments).

Example:

Car myCar = new Car();

Instance Variables

Instance variables are variables declared within a class but outside any method. Each object of the class has its own copy of the instance variables.

Example:

public class Car {
                String brand;
                int year;
            }

Methods

Methods are functions defined within a class. They define the behaviors of objects of the class.

Example:

public class Car {
                String brand;
                int year;
            
                public void drive() {
                    System.out.println("Driving...");
                }
            }

Accessing Class Members

You can access instance variables and methods of a class using the dot (.) operator with the object name.

Example:

Car myCar = new Car();
            myCar.brand = "Toyota";
            myCar.year = 2022;
            myCar.drive(); // Calls the drive method

Constructor

A constructor is a special method used to initialize objects. It has the same name as the class and no return type.

Example:

public class Car {
                String brand;
                int year;
            
                // Constructor
                public Car(String b, int y) {
                    brand = b;
                    year = y;
                }
            }
            
            // Creating an object with constructor arguments
            Car myCar = new Car("Toyota", 2022);

Instance vs. Static Members

Instance members (variables and methods) belong to individual objects, whereas static members belong to the class itself.

Example:

public class Car {
                static int count;
                String brand;
            
                public Car(String b) {
                    brand = b;
                    count++;
                }
            }

In this example, count is a static variable that keeps track of the number of cars created.

Inheritance

In Java, inheritance is a mechanism in which one class inherits the properties and behaviors (methods) of another class. The class that inherits is called the subclass or derived class, and the class that is inherited from is called the superclass or base class.

Superclass and Subclass

To establish inheritance, you use the extends keyword to indicate that a class is a subclass of another class.

Example:

// Superclass
            public class Animal {
                public void eat() {
                    System.out.println("Eating...");
                }
            }
            
            // Subclass inheriting from Animal
            public class Dog extends Animal {
                public void bark() {
                    System.out.println("Barking...");
                }
            }

Accessing Superclass Members

A subclass can access the members (fields and methods) of its superclass using the super keyword.

Example:

public class Dog extends Animal {
                public void bark() {
                    System.out.println("Barking...");
                    super.eat(); // Call eat method of superclass
                }
            }

Method Overriding

Subclasses can provide their own implementation for the methods inherited from the superclass. This is known as method overriding.

Example:

// Superclass
            public class Animal {
                public void makeSound() {
                    System.out.println("Animal makes a sound");
                }
            }
            
            // Subclass overriding makeSound method
            public class Dog extends Animal {
                @Override
                public void makeSound() {
                    System.out.println("Dog barks");
                }
            }

Constructors in Inheritance

When a subclass object is created, the constructor of both the superclass and subclass are called. If the superclass has a parameterized constructor, the subclass constructor must explicitly call it using the super keyword.

Example:

// Superclass with parameterized constructor
            public class Animal {
                public Animal(String type) {
                    System.out.println("Animal type: " + type);
                }
            }
            
            // Subclass calling superclass constructor
            public class Dog extends Animal {
                public Dog(String type) {
                    super(type); // Call superclass constructor
                    System.out.println("Dog type: " + type);
                }
            }

Polymorphism

In Java, polymorphism refers to the ability of a reference variable to hold objects of multiple types, and the method that gets called is determined by the object's type at runtime. There are two types of polymorphism in Java: compile-time (static) polymorphism and runtime (dynamic) polymorphism.

Compile-Time Polymorphism

Compile-time polymorphism is achieved through method overloading, where multiple methods with the same name but different parameters are defined within the same class.

Example:

public class Calculator {
                // Method overloading
                public int add(int a, int b) {
                    return a + b;
                }
            
                public double add(double a, double b) {
                    return a + b;
                }
            }

Runtime Polymorphism

Runtime polymorphism is achieved through method overriding, where a method in a subclass has the same signature (name and parameters) as a method in its superclass.

Example:

// Superclass
            public class Animal {
                public void makeSound() {
                    System.out.println("Animal makes a sound");
                }
            }
            
            // Subclass overriding makeSound method
            public class Dog extends Animal {
                @Override
                public void makeSound() {
                    System.out.println("Dog barks");
                }
            }

Polymorphic Behavior

Polymorphic behavior allows a reference variable of a superclass to refer to objects of its subclasses, and the appropriate method is called based on the actual object type at runtime.

Example:

Animal myAnimal = new Dog();
            myAnimal.makeSound(); // Calls makeSound method of Dog class

In this example, the makeSound method of the Dog class is called even though the reference variable is of type Animal. This is polymorphic behavior.

Abstraction

In Java, abstraction is the concept of hiding the complex implementation details and showing only the essential features of an object. Abstraction can be achieved through abstract classes and interfaces.

Abstract Classes

An abstract class is a class that cannot be instantiated and may contain abstract methods, which are declared but not implemented in the abstract class. Abstract methods must be implemented by subclasses.

Example:

// Abstract class
            public abstract class Shape {
                // Abstract method
                public abstract double area();
            }
            
            // Concrete subclass implementing abstract method
            public class Circle extends Shape {
                private double radius;
            
                public Circle(double radius) {
                    this.radius = radius;
                }
            
                @Override
                public double area() {
                    return Math.PI * radius * radius;
                }
            }

Interfaces

An interface is a reference type similar to a class that contains only abstract methods, constants, and default methods. Classes can implement interfaces to provide specific behavior.

Example:

// Interface
            public interface Vehicle {
                void start();
                void accelerate();
                void stop();
            }
            
            // Class implementing interface
            public class Car implements Vehicle {
                @Override
                public void start() {
                    System.out.println("Car started");
                }
            
                @Override
                public void accelerate() {
                    System.out.println("Car accelerating");
                }
            
                @Override
                public void stop() {
                    System.out.println("Car stopped");
                }
            }

Benefits of Abstraction

  • Encapsulation: Abstraction helps in hiding the internal details and representing only the essential features of an object.
  • Code Reusability: Abstract classes and interfaces promote code reusability by allowing multiple classes to implement common behaviors.
  • Flexibility: Abstraction provides flexibility in design by allowing changes to be made in the implementation without affecting the code that uses the abstraction.

Encapsulation

In Java, encapsulation is the mechanism of wrapping the data (variables) and code (methods) together as a single unit. It involves bundling the data and methods that operate on the data within a class and controlling access to them through access modifiers.

Access Modifiers

Java provides four access modifiers to control the access level of classes, variables, constructors, and methods:

  • private: Accessible only within the same class.
  • default (no modifier): Accessible only within the same package.
  • protected: Accessible within the same package and subclasses (even if they are in a different package).
  • public: Accessible from anywhere.

Encapsulation with Getters and Setters

Encapsulation is often achieved by providing getter and setter methods to access and modify the private data members of a class, respectively.

Example:

public class Person {
                private String name;
                private int age;
            
                // Getter method for name
                public String getName() {
                    return name;
                }
            
                // Setter method for name
                public void setName(String name) {
                    this.name = name;
                }
            
                // Getter method for age
                public int getAge() {
                    return age;
                }
            
                // Setter method for age
                public void setAge(int age) {
                    if (age >= 0) {
                        this.age = age;
                    } else {
                        System.out.println("Invalid age");
                    }
                }
            }

Benefits of Encapsulation

  • Data Hiding: Encapsulation hides the internal details of a class from outside interference, preventing unauthorized access and modification of data.
  • Controlled Access: Encapsulation allows access to data only through specified methods (getters and setters), enabling better control over data manipulation.
  • Modularity: Encapsulation promotes modularity by grouping related data and methods together within a class, making the code more organized and easier to maintain.

Interfaces

In Java, an interface is a reference type similar to a class that contains only abstract methods, constants, and default methods. It defines a contract for classes to implement, specifying a set of methods that implementing classes must provide.

Interface Declaration

To declare an interface in Java, you use the interface keyword followed by the interface name.

Example:

// Interface declaration
            public interface Drawable {
                void draw(); // Abstract method
            }

Implementing an Interface

Classes can implement interfaces by using the implements keyword, and they must provide implementations for all the abstract methods declared in the interface.

Example:

// Class implementing the Drawable interface
            public class Circle implements Drawable {
                @Override
                public void draw() {
                    System.out.println("Drawing a circle");
                }
            }

Interface Constants

Interfaces can contain constants, which are implicitly public, static, and final.

Example:

// Interface with constant
            public interface Constants {
                int MAX_VALUE = 100;
            }

Default Methods

Java 8 introduced default methods in interfaces, allowing interface methods to have a default implementation. Classes implementing the interface can use the default implementation or override it if needed.

Example:

// Interface with default method
            public interface Vehicle {
                void start();
            
                default void stop() {
                    System.out.println("Vehicle stopped");
                }
            }

Error Handling

In Java, error handling refers to the process of managing exceptions that occur during the execution of a program. Exceptions are objects that represent exceptional conditions that can occur during the execution of a program.

Types of Exceptions

There are two main types of exceptions in Java:

  • Checked Exceptions: Checked exceptions are checked at compile time and must be caught or declared in the method signature using the throws keyword.
  • Unchecked Exceptions (Runtime Exceptions): Unchecked exceptions are not checked at compile time and can be caught or left to propagate up the call stack.

Exception Handling

Exception handling in Java is done using try-catch blocks. The try block contains the code that may throw an exception, and the catch block catches and handles the exception.

Example:

try {
                // Code that may throw an exception
                int result = 10 / 0; // This will throw ArithmeticException
            } catch (ArithmeticException e) {
                // Exception handling
                System.out.println("An error occurred: " + e.getMessage());
            }

Finally Block

The finally block is used to execute code that should always be executed, regardless of whether an exception occurs or not. It is often used for cleanup operations such as closing resources.

Example:

try {
                // Code that may throw an exception
                // ...
            } catch (Exception e) {
                // Exception handling
                // ...
            } finally {
                // Cleanup code
                // ...
            }

Throwing Exceptions

You can manually throw exceptions using the throw keyword. This is useful for signaling exceptional conditions in your code.

Example:

public void withdraw(double amount) throws InsufficientBalanceException {
                if (amount > balance) {
                    throw new InsufficientBalanceException("Insufficient balance");
                }
                // Withdraw logic
            }

File Handling

In Java, file handling refers to reading from and writing to files on the file system. Java provides several classes and interfaces in the java.io package for working with files.

Reading from a File

To read from a file in Java, you typically use FileInputStream or BufferedReader classes. You open a file, read its contents, and then close the file.

Example using BufferedReader:

import java.io.BufferedReader;
            import java.io.FileReader;
            import java.io.IOException;
            
            public class ReadFile {
                public static void main(String[] args) {
                    try (BufferedReader reader = new BufferedReader(new FileReader("filename.txt"))) {
                        String line;
                        while ((line = reader.readLine()) != null) {
                            System.out.println(line);
                        }
                    } catch (IOException e) {
                        e.printStackTrace();
                    }
                }
            }

Writing to a File

To write to a file in Java, you typically use FileOutputStream or BufferedWriter classes. You open a file, write data to it, and then close the file.

Example using BufferedWriter:

import java.io.BufferedWriter;
            import java.io.FileWriter;
            import java.io.IOException;
            
            public class WriteFile {
                public static void main(String[] args) {
                    try (BufferedWriter writer = new BufferedWriter(new FileWriter("filename.txt"))) {
                        writer.write("Hello, world!");
                    } catch (IOException e) {
                        e.printStackTrace();
                    }
                }
            }

Working with Directories

You can also create, delete, and manipulate directories in Java using the File class.

Example:

import java.io.File;
            
            public class DirectoryExample {
                public static void main(String[] args) {
                    File directory = new File("directoryName");
                    if (!directory.exists()) {
                        if (directory.mkdir()) {
                            System.out.println("Directory created successfully");
                        } else {
                            System.out.println("Failed to create directory");
                        }
                    }
                }
            }

Generics

In Java, generics provide a way to create classes, interfaces, and methods that operate on objects of specified types (parameters) without having to define the actual type until the time of use. Generics enable you to write more flexible and reusable code.

Generic Classes

A generic class is a class that has one or more type parameters enclosed in angle brackets (< and >). These type parameters can be used as placeholders for the actual types when the class is used.

Example:

public class Box<T> {
                private T value;
            
                public void setValue(T value) {
                    this.value = value;
                }
            
                public T getValue() {
                    return value;
                }
            }

Using Generic Classes

To use a generic class, you specify the actual type when creating an object of the class.

Example:

Box<Integer> intBox = new Box<>();
            intBox.setValue(10);
            int value = intBox.getValue(); // No casting required

Generic Methods

Similar to generic classes, you can also define generic methods that have their own type parameters.

Example:

public class Util {
                public <T> T add(T a, T b) {
                    return a + b; // This won't compile for all types T
                }
            }

Generic Interfaces

Interfaces can also be generic, allowing you to create more flexible interfaces that can work with different types.

Example:

public interface List<E> {
                void add(E element);
                E get(int index);
            }

Generic Wildcards

Generics also support wildcard types, which are used to represent an unknown type or a specific type that is a supertype of all other types.

Example:

public void printList(List<? extends Number> list) {
                for (Number n : list) {
                    System.out.println(n);
                }
            }

Java Collections Framework

In Java, the Collections Framework provides a set of interfaces and classes for working with collections of objects. It includes various types of collections such as lists, sets, queues, and maps, along with algorithms for manipulating them.

Collection Interfaces

The core interfaces in the Collections Framework are:

  • List: Ordered collection of elements with duplicates allowed (e.g., ArrayList, LinkedList).
  • Set: Unordered collection of unique elements (e.g., HashSet, TreeSet).
  • Queue: Collection used to hold elements prior to processing (e.g., PriorityQueue, LinkedList).
  • Map: Key-value pairs where each key is unique (e.g., HashMap, TreeMap).

Using Collections

To use collections, you create objects of the appropriate classes and use their methods to add, remove, and manipulate elements.

Example of using ArrayList:

import java.util.ArrayList;
            import java.util.List;
            
            public class Main {
                public static void main(String[] args) {
                    List<String> list = new ArrayList<>();
                    list.add("Java");
                    list.add("Python");
                    list.add("C++");
            
                    System.out.println("Elements in the list:");
                    for (String element : list) {
                        System.out.println(element);
                    }
                }
            }

Sorting Collections

You can sort collections using the sort() method of the Collections class, passing the collection as an argument.

Example:

import java.util.ArrayList;
            import java.util.Collections;
            import java.util.List;
            
            public class Main {
                public static void main(String[] args) {
                    List<Integer> numbers = new ArrayList<>();
                    numbers.add(5);
                    numbers.add(2);
                    numbers.add(8);
            
                    Collections.sort(numbers);
            
                    System.out.println("Sorted numbers:");
                    for (int num : numbers) {
                        System.out.println(num);
                    }
                }
            }

Iterating Over Collections

You can iterate over collections using iterators, enhanced for loops, or forEach method (introduced in Java 8).

Example:

List<String> names = new ArrayList<>();
            // Add elements to the list
            // Iterate using enhanced for loop
            for (String name : names) {
                System.out.println(name);
            }
            
            // Iterate using iterator
            Iterator<String> iterator = names.iterator();
            while (iterator.hasNext()) {
                String name = iterator.next();
                System.out.println(name);
            }
            
            // Iterate using forEach method
            names.forEach(name -> System.out.println(name));

Concurrency in Java

Concurrency in Java refers to the ability of a program to perform multiple tasks simultaneously. Java provides several mechanisms for concurrent programming, including threads, executors, and synchronization.

Threads

A thread is the smallest unit of execution within a process. In Java, you can create and manage threads using the Thread class or by implementing the Runnable interface and passing it to a Thread object.

Example using Thread class:

class MyThread extends Thread {
                public void run() {
                    // Code to be executed in a separate thread
                }
            }
            
            // Create and start a thread
            MyThread thread = new MyThread();
            thread.start();

Example using Runnable interface:

class MyRunnable implements Runnable {
                public void run() {
                    // Code to be executed in a separate thread
                }
            }
            
            // Create a Runnable object and pass it to a Thread object
            Thread thread = new Thread(new MyRunnable());
            thread.start();

Executors

The java.util.concurrent package provides the Executor framework for executing tasks asynchronously in a thread pool. It includes various executor implementations such as ExecutorService, ScheduledExecutorService, and ForkJoinPool.

Example using ExecutorService:

ExecutorService executor = Executors.newFixedThreadPool(5);
            executor.submit(() -> {
                // Task to be executed asynchronously
            });
            executor.shutdown(); // Shutdown the executor after use

Synchronization

Concurrency issues such as race conditions and data inconsistency can arise when multiple threads access shared resources concurrently. Synchronization in Java prevents such issues by restricting access to shared resources using synchronized blocks or methods.

Example using synchronized method:

class Counter {
                private int count;
            
                public synchronized void increment() {
                    count++;
                }
            }

Example using synchronized block:

class Counter {
                private int count;
            
                public void increment() {
                    synchronized (this) {
                        count++;
                    }
                }
            }

Networking in Java

Networking in Java refers to the ability to create networked applications that communicate over the Internet or an intranet. Java provides a rich set of classes and APIs for building networked applications, including support for sockets, URLs, and HTTP connections.

Socket Programming

Socket programming in Java enables communication between two endpoints over a network. Java supports both TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) sockets.

Example of TCP client:

import java.io.*;
            import java.net.*;
            
            public class TCPClient {
                public static void main(String[] args) {
                    try {
                        Socket socket = new Socket("hostname", 1234);
                        PrintWriter out = new PrintWriter(socket.getOutputStream(), true);
                        BufferedReader in = new BufferedReader(new InputStreamReader(socket.getInputStream()));
                        
                        out.println("Hello, server");
                        String response = in.readLine();
                        System.out.println("Server response: " + response);
                        
                        socket.close();
                    } catch (IOException e) {
                        e.printStackTrace();
                    }
                }
            }

Example of TCP server:

import java.io.*;
            import java.net.*;
            
            public class TCPServer {
                public static void main(String[] args) {
                    try {
                        ServerSocket serverSocket = new ServerSocket(1234);
                        System.out.println("Server started");
            
                        Socket clientSocket = serverSocket.accept();
                        PrintWriter out = new PrintWriter(clientSocket.getOutputStream(), true);
                        BufferedReader in = new BufferedReader(new InputStreamReader(clientSocket.getInputStream()));
            
                        String inputLine;
                        while ((inputLine = in.readLine()) != null) {
                            System.out.println("Client: " + inputLine);
                            out.println("Message received: " + inputLine);
                        }
            
                        clientSocket.close();
                        serverSocket.close();
                    } catch (IOException e) {
                        e.printStackTrace();
                    }
                }
            }

Database Connectivity in Java

Database connectivity in Java allows Java applications to interact with databases for storing, retrieving, and manipulating data. Java provides the JDBC (Java Database Connectivity) API for connecting Java applications with databases.

Connecting to a Database

To connect to a database using JDBC, you need to load the appropriate database driver and establish a connection using a connection URL, username, and password.

Example:

import java.sql.Connection;
            import java.sql.DriverManager;
            import java.sql.SQLException;
            
            public class DBConnection {
                public static void main(String[] args) {
                    String url = "jdbc:mysql://localhost:3306/mydatabase";
                    String username = "username";
                    String password = "password";
            
                    try {
                        Connection connection = DriverManager.getConnection(url, username, password);
                        System.out.println("Connected to the database");
                        // Perform database operations...
                        connection.close(); // Close the connection when done
                    } catch (SQLException e) {
                        e.printStackTrace();
                    }
                }
            }

Executing SQL Queries

Once connected, you can execute SQL queries using Statement or PreparedStatement objects.

Example of executing a SQL query:

import java.sql.Connection;
            import java.sql.DriverManager;
            import java.sql.ResultSet;
            import java.sql.SQLException;
            import java.sql.Statement;
            
            public class DBQuery {
                public static void main(String[] args) {
                    String url = "jdbc:mysql://localhost:3306/mydatabase";
                    String username = "username";
                    String password = "password";
            
                    try {
                        Connection connection = DriverManager.getConnection(url, username, password);
                        Statement statement = connection.createStatement();
                        ResultSet resultSet = statement.executeQuery("SELECT * FROM users");
                        
                        while (resultSet.next()) {
                            String name = resultSet.getString("name");
                            int age = resultSet.getInt("age");
                            System.out.println("Name: " + name + ", Age: " + age);
                        }
                        
                        resultSet.close();
                        statement.close();
                        connection.close();
                    } catch (SQLException e) {
                        e.printStackTrace();
                    }
                }
            }

Handling Transactions

JDBC supports transactions for executing a series of database operations as a single unit of work. You can control transactions using the Connection object's commit() and rollback() methods.

Example:

import java.sql.Connection;
            import java.sql.DriverManager;
            import java.sql.SQLException;
            
            public class DBTransaction {
                public static void main(String[] args) {
                    String url = "jdbc:mysql://localhost:3306/mydatabase";
                    String username = "username";
                    String password = "password";
            
                    try {
                        Connection connection = DriverManager.getConnection(url, username, password);
                        connection.setAutoCommit(false); // Disable auto-commit
                        
                        // Perform database operations...
                        
                        connection.commit(); // Commit the transaction if successful
                        connection.close();
                    } catch (SQLException e) {
                        e.printStackTrace();
                    }
                }
            }
